The Roblox Dumper scans memory and extracts offsets for various Roblox classes. Here's everything that gets dumped:

## Core Classes

### Instance

The base class for all Roblox objects. See [Roblox Instance documentation](https://create.roblox.com/docs/reference/engine/classes/Instance).

- Class name and descriptor
- Parent and children offsets
- Name property
- Attributes

### DataModel

The root of the Roblox game hierarchy. See [Roblox DataModel documentation](https://create.roblox.com/docs/reference/engine/classes/DataModel).

- Game ID, Place ID, Creator ID
- JobId (string) - UUID v4 format server instance identifier (e.g., `b601201e-4867-45eb-b477-284a80d227a0`)
- ServerIP (string) - Server IP address and port in format `IP|PORT` (e.g., `127.0.0.1|50752`)
- GameLoaded (uint32_t) - Game loading state indicator
- Workspace reference
- RunService, UserInputService, ClientReplicator

### MouseService

Mouse input handling service. See [Roblox MouseService documentation](https://create.roblox.com/docs/reference/engine/classes/MouseService).

- InputObject reference
- Position (Vector2) - Current mouse position in client coordinates (stored inside InputObject)

**Note:** During the MouseService phase, the dumper will automatically move the mouse cursor to different positions to scan for the Position offset. Please do not move your mouse during this phase or it will fail.

### Workspace

The main game world container. See [Roblox Workspace documentation](https://create.roblox.com/docs/reference/engine/classes/Workspace).

- Camera reference
- Gravity

### Humanoid

**Note:** The Color property is stored as a BrickColor enum value. To read and interpret BrickColor values, refer to the [Roblox BrickColor documentation](https://create.roblox.com/docs/reference/engine/enums/BrickColor).

## UI Classes

### GuiObject

Base class for 2D GUI elements like Frames, TextLabels, etc. See [Roblox GuiObject documentation](https://create.roblox.com/docs/reference/engine/classes/GuiObject).

- AnchorPoint (Vector2) - Two floats (x, y)
- BackgroundTransparency (float)
- BackgroundColor3 (Color3/RGB) - Three floats (r, g, b) in 0.0-1.0 range
- BorderSizePixel (uint32_t/int)
- BorderColor3 (Color3/RGB) - Three floats (r, g, b) in 0.0-1.0 range
- LayoutOrder (uint32_t/int)
- Position (UDim2) - Complex structure (see below)
- Rotation (float)
- Size (UDim2) - Complex structure (see below)
- ZIndex (uint32_t/int)

**UDim2 Structure:**
UDim2 is a two-dimensional value where each dimension has both a relative scale and absolute offset:

- **X.Scale** (float) - Relative X position/size (0.0 = left/0%, 1.0 = right/100%)
- **X.Offset** (int) - Absolute X position/size in pixels
- **Y.Scale** (float) - Relative Y position/size (0.0 = top/0%, 1.0 = bottom/100%)
- **Y.Offset** (int) - Absolute Y position/size in pixels

Memory layout: `[X.Scale (float), X.Offset (int), Y.Scale (float), Y.Offset (int)]` - 16 bytes total

For more information on UDim2, see [Roblox UDim2 documentation](https://create.roblox.com/docs/reference/engine/datatypes/UDim2).

### UIGradient

UI gradient effect for GUI elements. See [Roblox UIGradient documentation](https://create.roblox.com/docs/reference/engine/classes/UIGradient).

- Color (Color3) - Stored as normalized floats (0.0-1.0 range)
- Offset
- Rotation
- Transparency

### TextLabel

Text rendering GUI element. See [Roblox TextLabel documentation](https://create.roblox.com/docs/reference/engine/classes/TextLabel).

- ContentText (string) - Read-only, actual text content displayed
- Text (string)
- LineHeight (float) - Line spacing multiplier
- MaxVisibleGraphemes (uint32_t/int) - Maximum visible characters
- TextBounds (Vector2) - Read-only, rendered text size in pixels (X = width, Y = height)
- TextStrokeTransparency (float) - Text outline transparency (0.0-1.0)
- TextTransparency (float) - Text transparency (0.0-1.0)
- TextColor3 (Color3/RGB) - Three floats (r, g, b) in 0.0-1.0 range
- TextStrokeColor3 (Color3/RGB) - Text outline color, three floats (r, g, b) in 0.0-1.0 range
- TextSize (int32_t) - Font size in pixels

### Tool

Player-equippable tool objects. See [Roblox Tool documentation](https://create.roblox.com/docs/reference/engine/classes/Tool).

- CanBeDropped (bool/uint8_t) - Whether the tool can be dropped
- Enabled (bool/uint8_t) - Whether the tool is enabled
- ManualActivationOnly (bool/uint8_t) - Whether tool requires manual activation
- RequiresHandle (bool/uint8_t) - Whether tool requires a handle part
- Grip (CFrame) - The grip CFrame (base offset)
- GripPos (Vector3) - The position component of the grip
- GripRight (Vector3) - The right vector of the grip orientation
- GripUp (Vector3) - The up vector of the grip orientation
- GripForward (Vector3) - The forward vector of the grip orientation
- ToolTip (string) - The tooltip text displayed when hovering over the tool

**Note:** The Grip CFrame contains the position and orientation components. GripPos, GripRight, GripUp, and GripForward are all derived from the Grip CFrame for easier access to individual components.

## Scripts

### ModuleScript

Module scripts that can be required by other scripts. See [Roblox ModuleScript documentation](https://create.roblox.com/docs/reference/engine/classes/ModuleScript).

- Bytecode (uintptr_t) - Pointer to embedded bytecode
- Hash (int32_t) - Script hash identifier

### LocalScript

Client-side scripts that run on the local client. See [Roblox LocalScript documentation](https://create.roblox.com/docs/reference/engine/classes/LocalScript).

- Bytecode (uintptr_t) - Pointer to embedded bytecode
- Hash (int32_t)

### ByteCode

Bytecode data structure used by both ModuleScript and LocalScript.

- Pointer (uintptr_t) - Offset to bytecode pointer
- Size (int32_t) - Size of the bytecode in bytes

## Values

### Value

Generic value objects. See [Roblox ValueBase documentation](https://create.roblox.com/docs/reference/engine/classes/ValueBase).

- Value property (for StringValue, NumberValue, etc.)

## Visual Engine

### VisualEngine

Low-level rendering engine.

- Pointer to VisualEngine
- RenderView
- ViewMatrix (4x4 matrix)
- WindowDimensions (Vector2)

### RenderView

Rendering view information.

- InvalidateLighting

### ProximityPrompt

Proximity prompt interaction objects. See [Roblox ProximityPrompt documentation](https://create.roblox.com/docs/reference/engine/classes/ProximityPrompt).

- **ActionText** (RobloxString) - Text shown for the action (e.g., "TestAction123")
- **ObjectText** (RobloxString) - Object label text (e.g., "TestObject456")
- **HoldDuration** (float) - Time to hold interaction (unique value: 2.56)
- **KeyboardKeyCode** (int32_t) - Keyboard key enum value (E = 101)
- **MaxActivationDistance** (float) - Maximum interaction distance (12.5)
- **Enabled** (bool/uint8_t) - Whether prompt is enabled
- **RequiresLineOfSight** (bool/uint8_t) - Requires line of sight to activate

## Output

All dumped offsets are saved to `offsets.hpp` in a structured format:

```cpp
namespace offsets {
    namespace Instance {
        uintptr_t ClassDescriptor = 0x...;
        uintptr_t Parent = 0x...;
        // ... more offsets
    }
    namespace Part {
        uintptr_t Position = 0x...;
        uintptr_t Size = 0x...;
        // ... more offsets
    }
    // ... more classes
}
```

All offsets are saved to `offsets.hpp` and can be used in your C++ projects to read memory from Roblox games.
